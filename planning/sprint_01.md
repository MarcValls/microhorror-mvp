# Sprint 01

## Objetivo
Levantar el vertical slice mínimo que pruebe la arquitectura y reduzca el riesgo técnico principal.

## Resultado esperado
Una persona del equipo puede crear un proyecto simple con una plantilla, ejecutar playtest local y completar una sesión básica.

## Historias incluidas
- login básico o modo invitado controlado
- creación de proyecto desde una plantilla
- 1 plantilla funcional
- 1 amenaza base
- 2 eventos base
- 1 objetivo simple
- 1 final
- guardado de borrador
- playtest local

## Tareas de producto
- cerrar definición del vertical slice
- fijar restricciones del editor
- acordar matriz de dispositivos objetivo

## Tareas de cliente
- shell de navegación (SceneTree + autoloads)
- escena base del runtime en primera persona
- panel mínimo del editor (presets y módulos)
- integración de configuración por datos (Resources de Godot)

## Tareas de backend
- esquema mínimo para User y Project
- guardado de borradores
- autenticación básica

## QA
- comprobar tiempos de carga
- validar flujo completo sin bloqueo
- revisar estabilidad en al menos dos dispositivos de referencia

## Criterio de cierre del sprint
El equipo puede demostrar en directo el flujo crear -> probar -> completar dentro de una única plantilla.
