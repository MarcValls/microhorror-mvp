# User flows

## Flow 1 — Creador
1. entra a la app
2. inicia sesión o continúa con fricción mínima
3. selecciona una plantilla
4. personaliza atmósfera, amenaza, eventos e historia
5. ejecuta playtest
6. ajusta configuración
7. publica
8. comparte enlace
9. revisa métricas básicas

## Flow 2 — Jugador
1. recibe un enlace
2. abre la experiencia
3. inicia sesión de juego
4. comprende el objetivo básico
5. completa la partida o falla
6. recibe pantalla de resultado
7. comparte resultado o rejuega

## Flow 3 — Soft launch interno
1. el equipo publica un proyecto
2. comparte con cohorte cerrada
3. observa tasa de apertura, inicio y finalización
4. identifica abandono por plantilla o dispositivo
5. itera sobre el contenido o el runtime
