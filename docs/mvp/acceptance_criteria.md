# Acceptance criteria del MVP

## Onboarding
- un usuario nuevo puede iniciar un proyecto en menos de 2 minutos
- el registro no bloquea el descubrimiento principal

## Plantillas
- existen 3 plantillas iniciales visibles y seleccionables
- cada plantilla muestra duración estimada y tono

## Editor
- el creador puede ajustar atmósfera, amenaza, historia, eventos y finales sin escribir código
- el creador puede iniciar playtest desde el mismo proyecto
- el proyecto se guarda como borrador

## Runtime
- la sesión carga en tiempo razonable en dispositivo objetivo
- existe al menos un objetivo claro
- existen al menos dos estados finales en el catálogo del MVP

## Publicación
- una experiencia válida genera un enlace único
- el enlace abre una sesión jugable o una entrada clara al juego
- la pantalla final incluye compartir resultado

## Analítica
- se registran los eventos principales de creación, publicación y sesión
- cada proyecto muestra métricas básicas agregadas

## Calidad
- no hay bloqueos críticos en creación, publicación o juego
- el rendimiento es aceptable en la matriz de dispositivos objetivo
- el contenido es configurable por datos, no por lógica hardcodeada proyecto a proyecto
